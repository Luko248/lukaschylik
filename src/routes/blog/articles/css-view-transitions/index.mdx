## Prechody medzi stavmi stránky natívne v prehliadači

Dlhé roky boli animované prechody medzi stránkami alebo stavmi aplikácie doménou JavaScript frameworkov. Či už šlo o Barba.js, Swup alebo vlastné riešenia cez GSAP — vždy to znamenalo extra kód, komplexnú logiku a problémy s prístupnosťou. CSS View Transitions API toto zásadne mení.

Prehliadač teraz dokáže sám zachytiť starý stav (snapshot), aplikovať DOM zmeny a animovať prechod na nový stav. A to buď čisto v CSS pri navigácii medzi stránkami (MPA), alebo s minimálnym JavaScriptom pri zmenách v rámci jednej stránky (SPA). Poďme si to rozobrať od základov.

## Ako View Transitions fungujú pod kapotou

Celý mechanizmus je založený na koncepte **snapshotov**:

1. Prehliadač zachytí **starý stav** (old snapshot) — vizuálny obraz aktuálneho stavu elementov
2. Vykoná sa zmena DOM (navigácia alebo mutácia)
3. Prehliadač zachytí **nový stav** (new snapshot)
4. Oba snapshoty sa vykreslia ako pseudo-elementy a prehliadač medzi nimi animuje prechod

Počas animácie sú snapshoty umiestnené vo **view transition overlay**, ktorý sa nachádza nad celým obsahom stránky. Používateľ vidí plynulý prechod, aj keď pod ním prebieha kompletná zmena DOM.

## Pseudo-elementy: Strom view transition

View Transitions API generuje špeciálny strom pseudo-elementov, ktorý môžeme štýlovať pomocou CSS:

```
::view-transition
└── ::view-transition-group(name)
    └── ::view-transition-image-pair(name)
        ├── ::view-transition-old(name)
        └── ::view-transition-new(name)
```

### Syntax selektorov

Všetky pseudo-elementy (okrem `::view-transition`) prijímajú parameter, ktorý identifikuje cieľový prechod:

```css
::view-transition-group(hero-image) { }       /* Podľa view-transition-name */
::view-transition-group(*.card) { }            /* Podľa view-transition-class (bodka pred názvom) */
::view-transition-group(hero-image.featured) { } /* Kombinácia name + class */
::view-transition-group(*) { }                 /* Wildcard — všetky pomenované prechody */
::view-transition-group(root) { }              /* Predvolená root skupina (celý dokument) */
```

### ::view-transition

Koreňový kontajner celého view transition overlay. Pokrýva celý viewport a je fixne pozicionovaný cez `inset: 0`. Všetky ostatné pseudo-elementy sú jeho children. Môžeme ho štýlovať, napríklad pridať polo-priehľadné pozadie:

```css
::view-transition {
  background-color: oklch(0% 0 0 / 0.25);
}
```

### ::view-transition-group()

Predstavte si, že máte na stránke hero obrázok — malý thumbnail v zozname a veľký na detaile. `::view-transition-group()` je pseudo-element, ktorý sa stará o to, aby sa tento obrázok plynule **presunul a zmenil veľkosť** z jednej pozície na druhú. Prehliadač automaticky vypočíta rozdiely v `width`, `height` a pozícii medzi starým a novým stavom a vygeneruje animáciu. Predvolená `animation-duration` je `0.25s`, ale môžete ju ľubovoľne prepísať.

### ::view-transition-image-pair()

Vnútri každej `group` sa nachádza `image-pair` — kontajner, ktorý drží starý a nový snapshot vedľa seba a zabezpečuje ich vzájomné prelínanie. Kľúčový detail je predvolený `mix-blend-mode: plus-lighter`. Prečo? Pri bežnom alpha blendovaní (keď starý obrázok mizne a nový sa objavuje) dochádza v strede animácie k viditeľnému stmavnutiu — oba sú polo-priehľadné a ich farby sa "strácajú". Režim `plus-lighter` namiesto toho ich svetelné hodnoty sčítava, takže cross-fade vyzerá vizuálne korektne bez tmavej medzifázy.

```css
/* Ak chcete štandardné blendovanie (napr. pre špeciálne efekty): */
::view-transition-image-pair(root) {
  isolation: auto;
}
```

### ::view-transition-old() a ::view-transition-new()

Toto sú samotné vizuálne snapshoty — `old` je statický screenshot toho, čo bolo na stránke pred prechodom (ako keby ste si urobili printscreen), a `new` je živá reprezentácia nového DOM stavu, ktorá sa aktualizuje v reálnom čase. Oba sa správajú ako **replaced elementy** (podobne ako `<img>`), takže na ne môžete aplikovať `object-fit` a `object-position` — užitočné keď sa veľkosť snapshotu líši od veľkosti cieľového pseudo-elementu.

Predvolené animácie sú jednoduché: `old` postupne mizne (fade-out, opacity 1 → 0) a `new` sa postupne objavuje (fade-in, opacity 0 → 1). Tieto animácie môžete kompletne nahradiť vlastnými `@keyframes`.

```css
::view-transition-old(hero),
::view-transition-new(hero) {
  /* Obsah vyplní celý pseudo-element a orezanie sa vykoná */
  object-fit: cover;
}
```

### Príklad štýlovania:

```css
/* Spoločné nastavenie pre všetky prechody */
::view-transition-group(*) {
  animation-duration: 0.4s;
  animation-timing-function: ease-in-out;
}

/* Starý stav sa odscrolluje hore */
::view-transition-old(root) {
  animation: slide-out-up 0.4s ease-in-out forwards;
}

/* Nový stav sa nascrolluje zdola */
::view-transition-new(root) {
  animation: slide-in-up 0.4s ease-in-out forwards;
}

@keyframes slide-out-up {
  to { translate: 0 -100%; opacity: 0; }
}

@keyframes slide-in-up {
  from { translate: 0 100%; opacity: 0; }
}
```

## CSS vlastnosti pre View Transitions

### view-transition-name

Najdôležitejšia vlastnosť. Priraďuje elementu identifikátor, podľa ktorého prehliadač vie, že starý a nový element patria k sebe a má medzi nimi animovať prechod.

```css
view-transition-name: none;            /* Predvolená hodnota — element sa nezúčastňuje */
view-transition-name: hero-image;      /* Vlastný identifikátor (<custom-ident>) */
view-transition-name: match-element;   /* Špeciálna hodnota — automatické párovanie */
```

**Dôležité pravidlo:** V rámci jedného scope musí byť každý `view-transition-name` s hodnotou `<custom-ident>` unikátny! Ak dva elementy zdieľajú rovnaký názov, `ViewTransition.ready` promise sa odmietne a prechod sa preskočí.

**Vedľajšie efekty:** Element s nenulovou hodnotou `view-transition-name` automaticky vytvára **stacking context** a je **sploštený v 3D transformáciách** (flattened). Tieto efekty sú aktívne vždy, nielen počas prechodu.

#### Hodnota `match-element`

Špeciálna hodnota `match-element` automaticky páruje elementy medzi starým a novým stavom na základe identity DOM uzla, nie na základe textového názvu. Je ideálna pre zoznamy, kde sa menia pozície položiek:

```css
.list-item {
  view-transition-name: match-element;
}
```

**Obmedzenie:** `match-element` funguje len pri same-document (SPA) transitions, nie pri cross-document navigácii, pretože identita DOM uzla sa medzi stránkami nezachováva.

### view-transition-class

Umožňuje zoskupiť viaceré pomenované prechody pod jednu triedu, aby sme im mohli nastaviť spoločné animácie. Bez tejto vlastnosti by sme museli písať pravidlá pre každý `view-transition-name` zvlášť.

**Dôležité:** Element stále potrebuje vlastný unikátny `view-transition-name`. Class slúži len ako **štýlovací hook** v pseudo-element selektoroch (pomocou bodkovej notácie):

```css
.card {
  view-transition-name: card-1;
  view-transition-class: card;
}

.card:nth-child(2) {
  view-transition-name: card-2;
  view-transition-class: card;
}

/* Spoločná animácia pre všetky karty — bodka pred názvom triedy */
::view-transition-group(*.card) {
  animation-duration: 0.3s;
}

::view-transition-old(*.card) {
  animation: shrink-out 0.3s ease-out both;
}

::view-transition-new(*.card) {
  animation: grow-in 0.3s ease-out both;
}
```

Element môže mať aj viacero tried:

```css
.product {
  view-transition-name: product-1;
  view-transition-class: card featured;
}
```

### view-transition-group

Kontroluje, kam sa `::view-transition-group()` pseudo-element zaradí v strome pseudo-elementov. Predvolene sú všetky skupiny priamymi children `::view-transition`. Táto vlastnosť umožňuje vytvárať **hierarchické vnorenie**:

```css
view-transition-group: normal;          /* Predvolená — priamy child ::view-transition */
view-transition-group: nearest;         /* Vnorí sa do ::view-transition-group-children() najbližšieho parent-u */
view-transition-group: contain;         /* Element sa stáva kontajnerom — jeho children
                                           budú obsiahnuté v rámci jeho scope-u */
view-transition-group: hero-container;  /* Vnorí sa do konkrétneho parent-u s daným view-transition-name */
```

#### Príklad hierarchického vnorenia:

```css
.container {
  view-transition-name: container;
  view-transition-group: contain; /* Povolí vnorenie children */
}

.container .item {
  view-transition-name: item-1;
  view-transition-group: nearest; /* Vnorí sa do kontajnera */
}
```

Výsledný pseudo-element strom:

```
::view-transition
└── ::view-transition-group(container)
    ├── ::view-transition-image-pair(container)
    │   ├── ::view-transition-old(container)
    │   └── ::view-transition-new(container)
    └── ::view-transition-group-children(container)
        └── ::view-transition-group(item-1)
            └── ::view-transition-image-pair(item-1)
                ├── ::view-transition-old(item-1)
                └── ::view-transition-new(item-1)
```

Toto je obzvlášť užitočné, keď chcete, aby sa children elementy animovali **relatívne voči parent kontajneru**, nie voči celému viewportu.

### view-transition-scope

Kontroluje viditeľnosť `view-transition-name` elementu v rámci stromu. Táto vlastnosť je kľúčová pre element-scoped transitions (viac nižšie):

```css
view-transition-scope: none;  /* Predvolená hodnota — normálne správanie */
view-transition-scope: all;   /* Názov je izolovaný — vonkajšie transitions ho preskočia */
```

## CSS-only View Transitions: @view-transition (MPA)

Tu sa dostávame k najzaujímavejšej časti pre tých, ktorí chcú animované prechody **bez jediného riadku JavaScriptu**. Pravidlo `@view-transition` umožňuje aktivovať view transitions pri navigácii medzi stránkami (cross-document navigation).

### Syntax

```css
@view-transition {
  navigation: auto;
}
```

Toto je všetko, čo potrebujete na oboch stránkach (zdrojovej aj cieľovej), aby prehliadač automaticky animoval prechod pri navigácii. Predvolená animácia je cross-fade.

### Deskriptor `navigation`

Najpraktickejšia hodnota je `auto`:

```css
@view-transition {
  navigation: auto;
}
```

Neznamená to však, že sa prechod spustí pri každej navigácii. Prehliadač ho aktivuje len vtedy, keď sú splnené tieto podmienky:

- Navigácia musí byť **same-origin**: obe stránky sú na rovnakom pôvode a medzi nimi nie je cross-origin presmerovanie.
- Typ navigácie musí byť jeden z týchto:
  - `traverse` = pohyb v histórii (späť/vpred)
  - `push` = pridanie nového záznamu do histórie (napr. klik na odkaz)
  - `replace` = nahradenie aktuálneho záznamu bez pridania nového
- Pri `push` a `replace` musí navigáciu vyvolať používateľ (klik, Enter, tap). Ak URL zmení skript na pozadí bez interakcie používateľa, prechod sa nespustí.

### Deskriptor `types`

Deskriptor `types` funguje ako sada štítkov pre konkrétny druh prechodu. Vďaka nim môžete jednej navigácii priradiť napríklad `slide` a inej `fade`, a následne pre každý typ napísať inú animáciu:

```css
@view-transition {
  navigation: auto;
  types: slide, fade;
}
```

Samotné `types` prechod nezapínajú. Len dávajú CSS informáciu, aký variant animácie sa má použiť.

Typy sa potom dajú cieliť pomocou pseudo-triedy `:active-view-transition-type()`:

```css
/* Animácia len pre typ "slide" */
:active-view-transition-type(slide) {
  &::view-transition-old(root) {
    animation: slide-out 0.4s ease-in-out both;
  }

  &::view-transition-new(root) {
    animation: slide-in 0.4s ease-in-out both;
  }
}
```

### Pseudo-trieda `:active-view-transition`

Cieli na koreňový element dokumentu počas aktívneho prechodu:

```css
/* Aplikuje sa počas akéhokoľvek aktívneho prechodu */
:active-view-transition {
  overflow: hidden; /* napr. skryť scrollbar počas prechodu */
}

/* Aplikuje sa len pre konkrétny typ */
:active-view-transition-type(slide) {
  /* ... */
}
```

### Kompletný MPA príklad (čisto CSS)

```css
/* === Obe stránky === */
@view-transition {
  navigation: auto;
}

/* Zdrojová stránka */
.hero-image { view-transition-name: hero; }
.page-title { view-transition-name: title; }

/* Cieľová stránka */
.detail-image { view-transition-name: hero; }
.detail-title { view-transition-name: title; }

/* Spoločné animácie */
::view-transition-old(root) {
  animation: fade-out 0.3s ease-out both;
}

::view-transition-new(root) {
  animation: fade-in 0.3s ease-out both;
}

::view-transition-group(hero) {
  animation-duration: 0.5s;
  animation-timing-function: cubic-bezier(0.4, 0, 0.2, 1);
}

@keyframes fade-out { to { opacity: 0; } }
@keyframes fade-in { from { opacity: 0; } }
```

V tomto príklade sa pri navigácii automaticky celá stránka plynule prejde cross-fade animáciou, hero obrázok sa presunie a zmení veľkosť, a titulok urobí to isté. **Žiadny JavaScript. Žiadna knižnica. Čisté CSS.**

## Kedy je CSS dostatočné a kedy potrebujete JavaScript?

Toto je kľúčová otázka. View Transitions API je navrhnuté tak, aby CSS pokrývalo čo najviac prípadov:

### CSS stačí (MPA prechody) keď:

- Animujete prechody pri **klasickej navigácii** medzi stránkami
- Používate **predvolené animácie** (cross-fade) alebo vlastné `@keyframes`
- Potrebujete **párovacie prechody** (hero image, titulky) medzi stránkami
- Chcete **rôzne typy animácií** podľa kontextu navigácie (cez `types`)

### JavaScript je nutný keď:

- Meníte DOM **v rámci jednej stránky** (SPA) — filtrovanie, triedenie, prepínanie tabov
- Potrebujete **programaticky preskočiť** prechod (`skipTransition()`)
- Chcete **reagovať na fázy prechodu** (kedy je snapshot hotový, kedy je animácia pripravená)
- Chcete spúšťať **viacero nezávislých prechodov** súčasne (element-scoped)
- Potrebujete **prispôsobiť MPA prechod** na cieľovej stránke cez `pagereveal` event

## JavaScript API: document.startViewTransition() (SPA)

Pre zmeny v rámci jednej stránky potrebujeme JavaScript. Metóda `document.startViewTransition()` obalí DOM mutáciu a prehliadač sa postará o animáciu:

```javascript
const transition = document.startViewTransition(() => {
  updateContent()
})
```

### Objekt ViewTransition

Metóda vracia objekt `ViewTransition` s tromi promise vlastnosťami, ktoré reprezentujú životný cyklus prechodu:

```javascript
const transition = document.startViewTransition(() => updateDOM())

// 1. DOM mutácia dokončená, snapshoty sa zachytávajú
transition.updateCallbackDone.then(() => console.log('DOM update hotový'))

// 2. Pseudo-elementy vytvorené, animácia pripravená na spustenie
transition.ready.then(() => console.log('Animácia pripravená'))

// 3. Celý prechod (vrátane animácie) dokončený
transition.finished.then(() => console.log('Prechod kompletne dokončený'))
```

### Preskočenie prechodu

Užitočné napríklad pre `prefers-reduced-motion`:

```javascript
const transition = document.startViewTransition(() => updateDOM())

if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
  transition.skipTransition()
}
```

### Vlastné animácie cez Web Animations API

Promise `ready` umožňuje nahradiť predvolené CSS animácie vlastnými cez JavaScript:

```javascript
const transition = document.startViewTransition(() => toggleDarkMode())

transition.ready.then(() => {
  // Kruhová animácia z pozície kliknutia
  document.documentElement.animate(
    {
      clipPath: [
        `circle(0% at ${x}px ${y}px)`,
        `circle(150% at ${x}px ${y}px)`,
      ],
    },
    {
      duration: 500,
      easing: 'ease-in-out',
      pseudoElement: '::view-transition-new(root)',
    }
  )
})
```

### SPA príklad: Filtrovanie zoznamu

```html
<ul class="items-grid" id="grid">
  <li class="item" data-category="design" style="view-transition-name: item-1">Dizajn systém</li>
  <li class="item" data-category="dev" style="view-transition-name: item-2">API architektúra</li>
</ul>
```

```javascript
function filterItems(category) {
  document.startViewTransition(() => {
    document.querySelectorAll('.item').forEach(item => {
      item.style.display =
        category === 'all' || item.dataset.category === category ? '' : 'none'
    })
  })
}
```

```css
.item { view-transition-class: grid-item; }

::view-transition-group(*.grid-item) {
  animation-duration: 0.3s;
  animation-timing-function: cubic-bezier(0.4, 0, 0.2, 1);
}
```

## MPA udalosti: pageswap a pagereveal

Pre cross-document navigáciu existujú dva špeciálne eventy, ktoré umožňujú prispôsobiť prechod pomocou JavaScriptu:

### pageswap (zdrojová stránka)

Vyvolá sa na stránke, z ktorej sa naviguje preč. Umožňuje upraviť prechod ešte pred opustením stránky:

```javascript
window.addEventListener('pageswap', (event) => {
  if (event.viewTransition) {
    console.log('Navigácia s prechodom prebiehá')
  }
})
```

### pagereveal (cieľová stránka)

Vyvolá sa na stránke, na ktorú sa naviguje. Umožňuje dynamicky prispôsobiť animáciu na cieľovej stránke:

```javascript
window.addEventListener('pagereveal', (event) => {
  if (event.viewTransition) {
    event.viewTransition.ready.then(() => {
      // Prispôsobenie animácie na základe kontextu
    })
  }
})
```

### Typy za behu (ViewTransitionTypeSet)

Typy prechodu je možné meniť aj dynamicky cez JavaScript:

```javascript
window.addEventListener('pagereveal', (event) => {
  if (event.viewTransition) {
    const fromUrl = new URL(navigation.activation.from)
    const toUrl = new URL(navigation.activation.entry.url)

    // Dynamické nastavenie typu podľa smeru navigácie
    if (isForwardNavigation(fromUrl, toUrl)) {
      event.viewTransition.types.add('slide-forward')
    } else {
      event.viewTransition.types.add('slide-backward')
    }
  }
})
```

## Element-Scoped View Transitions (Chrome 147+)

A teraz sa dostávame k najnovšej a najvýznamnejšej funkcionalite — **element-scoped view transitions**.

### Problém s document-scoped transitions

Doterajšie `document.startViewTransition()` má zásadné obmedzenia: len jeden prechod naraz, stránka je neinteraktívna počas prechodu, a problémy s `position: fixed` kvôli novému stacking context.

### Riešenie: Element.startViewTransition()

Nová metóda `startViewTransition()` na ľubovoľnom DOM elemente:

```javascript
element.startViewTransition({
  callback: () => {
    // DOM mutácie obmedzené na subtree tohto elementu
  }
})
```

Element sa stáva **scope root** — snapshotuje sa len jeho podstrom a zvyšok stránky zostáva plne interaktívny.

### Súčasné prechody

Viacero neprekrývajúcich sa DOM podstromov môže mať každý svoj vlastný view transition **v rovnakom čase**:

```javascript
// Sidebar filtruje nezávisle
sidebar.startViewTransition({ callback: () => updateFilters() })

// Grid sa preusporiadava nezávisle
grid.startViewTransition({ callback: () => reorderItems() })

// Detail panel sa aktualizuje nezávisle
detailPanel.startViewTransition({ callback: () => showDetail(itemId) })
```

Všetky tri animácie bežia súčasne a žiadna neblokuje ostatné.

### Znovupoužitie view-transition-name naprieč scope-mi

Toto je obrovské ergonomické zlepšenie. Predtým musel byť každý `view-transition-name` globálne unikátny v celom dokumente. Teraz môže rovnaký názov existovať vo viacerých scope-och bez konfliktu:

```css
/* Oba zoznamy môžu používať rovnaký transition name! */
#list-a li {
  view-transition-name: match-element;
  view-transition-class: album;
}

#list-b li {
  view-transition-name: match-element;
  view-transition-class: album;
}
```

### Automatické správanie scope root-u

Keď sa spustí element-scoped transition, prehliadač automaticky nastaví na scope root niekoľko pravidiel. V praxi to znamená:

<table class="w-full border-collapse border border-gray-200 dark:border-gray-800 bg-gray-100 dark:bg-gray-900 text-left">
  <thead class="bg-gray-200 dark:bg-gray-800/70 border-b border-gray-200 dark:border-gray-800">
    <tr>
      <th class="px-4 py-3">Automaticky aplikované pravidlo</th>
      <th class="px-4 py-3">Praktický efekt</th>
    </tr>
  </thead>
  <tbody>
    <tr class="odd:bg-gray-50 dark:odd:bg-gray-900 even:bg-gray-100 dark:even:bg-gray-800/70">
      <td class="px-4 py-3 border-b border-gray-200 dark:border-gray-800"><code>contain: layout</code></td>

```javascript
function moveTask(taskElement, targetColumnId) {
  const sourceColumn = taskElement.closest('.column')
  const targetColumn = document.getElementById(targetColumnId)

  // Oba stĺpce animujú súčasne
  sourceColumn.querySelector('ul').startViewTransition({
    callback: () => taskElement.remove()
  })

    callback: () => targetColumn.querySelector('ul').appendChild(taskElement)
```

## Prístupnosť a prefers-reduced-motion

View Transitions API rešpektuje používateľské preferencie. Vždy je dobré pridať fallback:

```css
@media (prefers-reduced-motion: reduce) {
  ::view-transition-group(*),
  ::view-transition-old(*),
  ::view-transition-new(*) {
    animation-duration: 0.01ms !important;
  }
}
```

Alternatívne v JavaScripte:

```javascript
function safeStartTransition(callback) {
  if (
    !document.startViewTransition ||
    window.matchMedia('(prefers-reduced-motion: reduce)').matches
  ) {
    callback()
    return
  }

  document.startViewTransition(callback)
}
```

## Výkon a Core Web Vitals

Poškodzujú view transitions výkon stránky? Krátka odpoveď — nie, ak ich používate správne.

### CLS (Cumulative Layout Shift)

View transitions **neovplyvňujú CLS**. Prechodové pseudo-elementy sa renderujú v top-layer overlaji, úplne oddelene od document flow. Samotná DOM zmena prebehne, kým je rendering potlačený — layout shift je pre používateľa aj pre meranie CLS neviditeľný.

### LCP (Largest Contentful Paint)

Reálne testy na 120 000 mobilných navigáciách ukázali, že view transitions pridávajú približne **70 ms k LCP na mobilných zariadeniach**. Na desktope s rýchlym CPU je overhead zanedbateľný (~5 ms). Čím pomalší procesor, tým väčší dopad — low-end Android zariadenia sú hlavnou oblasťou záujmu.

Kombinácia view transitions so **Speculation Rules API** (prerendering cieľových stránok) eliminuje LCP overhead úplne, pretože cieľová stránka je už vyrenderovaná keď prechod začne.

### INP (Interaction to Next Paint)

View transitions **priamo neovplyvňujú INP**. Prechod beží ako CSS animácia na kompozitných vrstvách. Ak však callback vykonáva náročné DOM mutácie, táto práca beží na main threade a môže nepriamo zvýšiť INP. Riešenie: udržujte update callback čo najľahší.

### GPU akcelerácia

Predvolená cross-fade animácia používa `transform` a `opacity` — dve CSS vlastnosti, ktoré sú plne kompozitné a GPU-akcelerované, obchádzajú layout aj paint. Chrome berie snapshot dáta priamo z compositor layeru bez dodatočných layout alebo repaint krokov. Vlastné animácie zachovajú túto výhodu, pokiaľ sa držíte `transform` a `opacity`.

### Hlavné riziká pre výkon

- **`blocking="render"`** — pri cross-document prechodoch oneskoruje FCP aj LCP. Používajte len keď aktívne meriete dopad na reálnych používateľoch.
- **Náročné DOM mutácie** v transition callbacku — presúvajte ťažkú prácu mimo callback.

### Odporúčanie

Vždy wrappujte `@view-transition` pravidlá do `prefers-reduced-motion: no-preference` media query. Rešpektujete tým prístupnosť a zároveň odstránite LCP penalty pre používateľov, ktorí majú vypnuté animácie.

```css
@media (prefers-reduced-motion: no-preference) {
  @view-transition {
    navigation: auto;
  }
}
```

## Podpora v prehliadačoch

Základné SPA view transitions (`document.startViewTransition()`) sú podporované v Chrome 111+, Firefox 132+ a Safari 18+. Cross-document MPA prechody (`@view-transition`) fungujú v Chrome 126+ a Safari 18.2+. Element-scoped transitions sú zatiaľ len v Chrome 147+.

View Transitions sú navrhnuté pre progresívne vylepšenie — ak prehliadač nepodporuje API, DOM zmena sa vykoná okamžite bez animácie:

```javascript
if (document.startViewTransition) {
  document.startViewTransition(() => updateDOM())
} else {
  updateDOM()
}
```

## Záver

View Transitions API je jedným z najvýznamnejších CSS/JS prírastkov posledných rokov. CSS-only MPA prechody cez `@view-transition` umožňujú vývojárom pridať plynulé animácie medzi stránkami bez akéhokoľvek JavaScriptu. Pre SPA aplikácie `document.startViewTransition()` poskytuje elegantné riešenie s minimálnym kódom. A nové element-scoped transitions v Chrome 147+ posúvajú možnosti ešte ďalej — viacero súčasných animácií, znovupoužiteľné názvy a plná interaktivita stránky počas prechodov.

Najdôležitejšie je, že celé API je navrhnuté pre progresívne vylepšenie. Ak prehliadač nepodporuje view transitions, stránka funguje úplne normálne — len bez animácií. Žiadne chyby, žiadne zlomené layouty. Preto nie je dôvod čakať — view transitions môžete začať používať už dnes.
